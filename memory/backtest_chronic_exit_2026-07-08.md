# Chronic-Never-Worked Early-Exit Backtest — 2026-07-08

Research-only diagnostic. No orders placed, no edits to `memory/strategy.md`. This is the H4 follow-up backtest called for in `memory/stop_audit_2026-07-07.md`.

Full technical report and per-trade CSV: `backtesting/reports/backtest_report_CHRONIC_EXIT.md` / `backtesting/reports/backtest_trades_CHRONIC_EXIT.csv`, generated by `backtesting/scripts/backtest_chronic_exit.py`. This file is the standalone summary of that run.

## 1. What was tested

`memory/stop_audit_2026-07-07.md` found that in the live 13-trade sample, chronic never-worked positions (majority of holding period closing underwater, MFE never confirmed +3%) were the modal hard-stop-loss bucket (4/8, 50%) — but 2 of those 4 (PWR, CASY) recovered fully after the eventual hard stop fired. H4 in that report called for a dedicated backtest weighing avoided chronic-drift losses against foreclosed recoveries before touching any live rule. This is that backtest.

**Rule tested:** on top of the current entry/exit rules (flat -8% hard stop, 1/3 scale-out + 7% trail at +10%, 42-calendar-day time stop), add one more exit trigger — exit at close if, by trading day D since entry, MFE (peak daily High since entry vs. entry price) has not reached +3% **and** the position has closed underwater for K consecutive sessions. The hard stop always takes priority (it's a resting order); the chronic-exit check only fires on days it hasn't already been stopped out.

**Grid (no cherry-picking, all 9 reported):** D ∈ {5, 7, 10} trading days, K ∈ {3, 4, 5} consecutive underwater closes.

**Dataset:** the same validated 2022–2025(+2026 YTD) candidate set used by every other PEAD backtest in this repo (`backtest_trades_PEAD_2022_2025_ENHANCED_base.csv` + `backtest_trades_PEAD_2026_YTD.csv`, enhanced-filter stack: surprise≥15%, volume, relative-strength, 52-week filters) — **277 trades across 182 tickers**, 2022-01-01 through 2026-07-04. No regime gate applied to the candidate set (see §6 Limitations — this matches the S3 ATR-stop test's own baseline, not the smaller 207-trade regime-gated sample, chosen for statistical power and for direct methodological comparability with the S3 test the user referenced). Price history fetched fresh from Yahoo's chart API for this run; 182/182 tickers resolved, 0 no-history/no-entry drops.

**Split discipline:** in-sample = 2022–2024 (194 trades), out-of-sample = 2025+2026 YTD (83 trades) — the same IS/OOS split used in `backtest_atr_stop_widened.py` (the S3 ATR-stop test), which is the split that correctly caught S3 failing OOS despite looking strong in-sample.

**The mechanism is symmetric and that's why it nets negative.** At every (D, K), the rule intercepts roughly as many eventual-winners (foreclosed) as it intercepts eventual hard-stop-losers (avoided) — e.g. D=5,K=3: 36 avoided vs. 41 foreclosed; D=10,K=3: 21 avoided vs. 24 foreclosed. The loss reduction per avoided trade (~+4.4 to +5.4pp, roughly halving a -9% loss to about -4%) is real and consistent, but the forgone gain per foreclosed trade (~+5.3 to +5.9pp, turning a real winner into a ~-2% loss) is comparable in magnitude and hits a similar or larger count of trades. This is a direct, larger-sample confirmation of the exact PWR/CASY tension flagged in the live 13-trade audit: MFE-not-yet-confirmed-by-day-D is not a reliable enough signal to separate "this one is dead" from "this one hasn't turned yet," at any tested (D, K).

## 6. Hypothesis verdict (H4, from the stop audit)

**H4 — "A chronic-position exit concept deserves testing before any wider-stop concept" — NOT SUPPORTED by this backtest, for any of the 9 tested (D, K) combinations.**

- *For (the audit's framing):* the concept correctly targets a real, large loss bucket — 27.1% of this backtest's full candidate population eventually hits the hard stop, and every combination intercepts a meaningful fraction of that bucket (28–48%) with a real, consistent loss reduction on the intercepted trades (~+4.4 to +5.4pp).
- *Against (this backtest's finding):* the same signal (MFE<3% + K underwater closes) that flags eventual hard-stop losers flags a comparable number of eventual winners, at every tested D and K. Net effect on full-sample average return, win rate, and profit factor is negative for all 9 combinations, and the OOS split (which is what actually validates a rule, per the S3 precedent) confirms the degradation holds out-of-sample too — none of the 9 combinations beats baseline OOS avg return or PF.
- **Verdict: reject the chronic-exit concept as tested.** This does not rule out a smarter version of the idea (e.g. a different signal than raw MFE/underwater-streak, a partial size-down instead of a full exit, or combining it with a catalyst/thesis check like the ones flagged in the original audit) — but the specific MFE+K-day-underwater trigger, at every D/K combination in this grid, is a net negative versus doing nothing.

## 7. Recommendation

**No change to `memory/strategy.md` is supported by this data.** None of the 9 combinations clears the out-of-sample bar; the mechanism analysis in §4 explains why (the signal doesn't separate dead trades from not-yet-turned trades cleanly enough to net positive after accounting for foreclosed recoveries). This closes out H4 from the stop audit as tested — a materially different signal design would be a new, separate proposal requiring its own fresh backtest, not a retest of this one.

## 8. Limitations

- **No regime gate applied to the candidate set** — matches the S3 ATR-stop script's own baseline (not the separate 207-trade regime-gated sample used for the bull-regime-gate variant reported elsewhere), chosen for statistical power (277 vs. ~207 trades) and direct comparability with the S3 methodology this task explicitly referenced. The live book currently trades under the regime gate, so this candidate population is broader than what the live strategy actually enters today — treat these results as evidence about the exit rule in isolation, not a like-for-like replay of the exact live entry population.
- **Survivorship bias**: tickers drawn from current S&P 500/400/600 constituents and current sector classifications, same caveat as every other backtest report in this repo.
- **MFE uses daily High, not full intraday path** — same simplification used in the live stop audit's multi-day trades.
- **Chronic-exit and hard-stop are both evaluated at the daily close** — assumes an end-of-day rule review executes at that close; a live analog would more likely execute at the next session's open (via `midday` or `market_close`), introducing modest unmodeled slippage.
- **2026 YTD sample is small** (30 candidate entries) — the OOS split's 2026 component carries limited individual weight; the OOS verdict is driven mostly by 2025 (53 trades).
- **This backtest answers a different question than the live 13-trade audit** — the audit's PWR/CASY recoveries are two specific real trades; this backtest's "foreclosed recovery" counts are the same *pattern* replayed across the full historical candidate set, not a claim that those exact two trades appear in this sample.
- No orders were placed and `memory/strategy.md` was not modified to produce this report.
